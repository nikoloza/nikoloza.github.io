inverse means column width that takes 100% - column size.

conspan(n) means column width that takes column * (n) size. For example, if column size is 10% for row10, colspan2 will be column size * 2 = 20%, colspan4 will be column size *4 = 40%.

large means grid size for 1920px and higher resolutions (imac)

big means grid size for 1440px and higher resolutions

small means grid size for 728px and lower resolutions (iphone landscape). default is 100% for each column.

tiny means grid size for 480px and lower resolutions (iphone portrait).

For example class="tiny1 small2 row10 big12 large16" means that your column width will be:

  • 100% for tiny screens
  • 50% for small screens
  • 10% for normal screens
  • 8.33(3)% for big (1440px+) screens.
  • 6.25% for big (1920px+) screens.
